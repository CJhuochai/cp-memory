每周执行一次 CP Memory 周维护，遵循“CP Memory 主库优先、治理安全优先”的原则。优先使用 CP Memory 的 MCP 工具。先调用 memory_health；再调用 memory_maintenance(dry_run=true, expire=false) 预览维护影响；再调用 memory_conflicts(limit=20) 和 memory_governance_report(limit=5) 检查冲突、复核队列、纠正样本与恢复探针；再调用 memory_auto_extract_cleanup(dry_run=true, limit=20, action="mark_wrong") 仅预览疑似噪声，不要在周维护中物理删除冲突记忆或自动提炼噪声。如果 memory_maintenance 的预览结果里 expired_candidates > 0，则执行 memory_maintenance(dry_run=false, expire=true)，只物理删除已过期、低重要性、未 pinned 的记忆。不要自动解决冲突。不要删除 high-importance、pinned、decision、task、workflow、alias、profile、preference、relationship、ongoing、belief_decision、active-task 或其他受治理保护的长期记忆。冲突项默认视为 stale/wrong/scoped 等治理对象，而不是删除对象。如果 CP Memory MCP 工具不可用，则只读检查默认 CP Memory 数据目录（例如 ~/.cp-memory/memory.db，或已配置的 CP_MEMORY_DB_PATH / CP_MEMORY_HOME），并在报告中说明由于 MCP 工具不可用，主动治理操作已跳过。输出健康指标、expired_candidates、deleted 数量、最值得处理的冲突、噪声预览候选，以及是否需要人工复核；只有在发生删除、存在冲突/复核队列、或出现健康异常时，才创建 inbox item，否则报告保持简洁。
